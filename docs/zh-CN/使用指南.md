# 中文使用指南

## 这是什么

本仓库是一套 Claude Code skills。每个 skill 都是一个独立的 `SKILL.md`，用于告诉 Claude Code 在特定任务中采用更合适的流程、检查点和输出方式。

这些 skills 不绑定具体技术栈，适合多数前后端项目、全栈项目、AI 项目、数据项目、脚本任务和办公任务。

当前版本包含 73 个 skills，并采用“窄 skill 优先”的设计：PPT 就只管 PPT，邮件就只管邮件，SQL 就只管 SQL。宽泛 skill 只在任务范围不清时作为总控/路由使用。

## 安装

### 安装到单个项目

把 `.claude` 目录复制到你的项目根目录：

```powershell
Copy-Item -Recurse .\.claude C:\path\to\your-project\
```

安装后：

```text
your-project/
  .claude/
    skills/
      <skill-name>/
        SKILL.md
```

### 安装到用户目录

如果你的 Claude Code 支持全局 skills，可复制到用户目录：

```powershell
New-Item -ItemType Directory -Force $HOME\.claude\skills
Copy-Item -Recurse .\.claude\skills\* $HOME\.claude\skills\
```

## 使用方法

Claude Code 会根据 `SKILL.md` 顶部的 `description` 判断是否使用某个 skill。你可以自然描述需求，不必每次点名 skill。

推荐写法：

```text
帮我实现这个登录接口，包含输入校验、错误处理和测试。
```

```text
请审查最近的改动，重点看安全、数据一致性和遗漏测试。
```

```text
把这份产品调研资料整理成一页决策简报。
```

显式触发写法：

```text
请使用 backend-api skill 设计这个订单接口。
```

```text
请使用 spreadsheet-analysis skill 清洗这个 CSV 并输出关键指标。
```

## 维护原则

- 每个 skill 保持短小，避免把通用编程知识写得过多。
- `description` 负责触发，必须明确写出使用场景。
- 正文只写执行流程、检查点和非显而易见的约束。
- 不在 skill 目录里放无关说明文档，GitHub 展示文档放在 `docs/`。
- 如果某个 skill 变得很长，优先拆分为更具体的 skill。

## 发布到 GitHub

建议提交以下内容：

```text
.claude/skills/
docs/zh-CN/
README.md
skills-directory.md
```

建议不要提交个人临时文件、系统生成文件、编辑器缓存和包含隐私信息的文档。
