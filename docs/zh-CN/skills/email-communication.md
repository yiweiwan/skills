# email-communication：邮件与沟通

## 文件位置

`.claude/skills/email-communication/SKILL.md`

## 适用场景

仅作为专业沟通任务的总控/路由 skill 使用。邮件和回复优先使用 `email-drafting`，状态更新优先使用 `stakeholder-update`，会议 recap 使用 `meeting-notes`，发布公告使用 `release-notes`。

## 能力重点

- 先识别沟通类型和受众。
- 能用更窄 skill 时不要使用本 skill。
- 明确负责人、截止日期、决策和下一步。
- 根据场景调整正式、温和、强硬或简洁程度。

## 示例提问

```text
请帮我写一封跟进客户反馈的邮件，语气专业但不要太强硬。
```

```text
把这段 Slack 消息改成适合发给老板的状态更新。
```

```text
帮我写一封项目延期说明邮件。
```
