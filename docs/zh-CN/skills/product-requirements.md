# product-requirements：产品需求

## 文件位置

`.claude/skills/product-requirements/SKILL.md`

## 适用场景

用于 PRD、用户故事、验收标准、产品规格、Feature Brief、需求澄清、范围决策、路线图条目、发布标准和干系人产品文档。

## 能力重点

- 明确用户、问题、目标、业务背景、约束和决策人。
- 区分目标、非目标、假设、需求和开放问题。
- 把需求写成可测试的验收标准。
- 纳入边界情况、权限、埋点、灰度、依赖和支持影响。

## 示例提问

```text
请把这个想法整理成一份轻量 PRD。
```

```text
帮我为这个功能写用户故事和验收标准。
```

```text
请检查这份需求是否有范围不清和遗漏边界条件。
```
