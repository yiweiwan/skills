# technical-writing：技术写作

## 文件位置

`.claude/skills/technical-writing/SKILL.md`

## 适用场景

仅作为技术写作类任务的总控/路由 skill 使用。README 使用 `readme-writing`，API 文档使用 `api-documentation`，运行手册使用 `runbook-writing`，架构文档使用 `architecture-docs`，发布说明使用 `release-notes`。

## 能力重点

- 明确受众、目标、前置条件和期望结果。
- 判断任务是 README、API 文档、Runbook、架构文档、发布说明还是混合文档。
- 能使用更窄 skill 时优先使用更窄 skill。
- 先查看真实代码、命令、API 和配置，再写文档。

## 示例提问

```text
请为这个项目补一份 README，包含启动、测试和部署。
```

```text
帮我写这个接口的 API 文档。
```

```text
请把这次升级整理成迁移指南。
```
