# code-review-refactor：代码审查与重构

## 文件位置

`.claude/skills/code-review-refactor/SKILL.md`

## 适用场景

仅作为代码质量任务的总控/路由 skill 使用。PR 或代码审查优先使用 `code-review`，保持行为不变的重构优先使用 `code-refactor`。

## 能力重点

- 先判断任务是审查、重构，还是两者都有。
- 审查任务只报告问题，不主动改代码，除非用户要求。
- 重构任务保持行为不变。
- 不把功能开发混进重构。

## 示例提问

```text
请审查这个 PR，重点看潜在回归和遗漏测试。
```

```text
帮我重构这个服务类，保持行为不变。
```

```text
请找出这段代码中可维护性最差的部分并修复。
```
