# debugging-troubleshooting：调试与故障排查

## 文件位置

`.claude/skills/debugging-troubleshooting/SKILL.md`

## 适用场景

用于诊断 bug、崩溃、命令失败、构建失败、运行时错误、性能问题、回归、日志、堆栈、CI 失败、依赖冲突和环境问题。

## 能力重点

- 先用最小命令或流程复现问题。
- 收集完整错误、堆栈、版本、环境和最近变更。
- 一次验证一个假设，避免盲改。
- 修复根因，并尽量补回归测试。

## 示例提问

```text
项目构建失败了，请根据错误日志排查原因。
```

```text
这个接口偶尔返回 500，请帮我定位根因。
```

```text
测试在 CI 里失败但本地通过，请分析可能原因。
```
