# testing-quality：测试与质量

## 文件位置

`.claude/skills/testing-quality/SKILL.md`

## 适用场景

用于添加、修复或改进测试、测试策略、覆盖率、mock、fixture、CI 失败、flaky test、单元测试、集成测试、端到端测试、视觉测试和可访问性检查。

## 能力重点

- 先识别测试框架、命名约定、fixture、mock 和 CI 命令。
- 先复现 bug 或定义行为，再改测试。
- 在最低但可靠的层级添加最小有用测试。
- 测试用户可观察行为和稳定契约。

## 示例提问

```text
请为这个接口补充单元测试和错误场景测试。
```

```text
这个测试偶发失败，请帮我定位并修复 flaky 原因。
```

```text
请为这个表单流程加一个端到端测试。
```
