# Project Planning

## Workflow

1. Identify goal, scope, stakeholders, constraints, deadline, dependencies, and definition of done.
2. Break work into deliverables that can be verified.
3. Sequence tasks by dependency and risk.
4. Surface assumptions, decisions needed, blockers, and unknowns.
5. Keep the plan actionable for the next person who will execute it.

## Patterns

- Implementation plan: context, approach, tasks, tests, rollout, rollback.
- Roadmap: outcomes, milestones, dependencies, sequencing, risks.
- Risk register: risk, likelihood, impact, mitigation, owner, trigger.

## Verification

- Check every task has an owner or owner role, output, and acceptance criteria.
- Confirm dependencies and critical path are visible.
