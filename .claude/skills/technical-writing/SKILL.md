# Technical Writing

## Workflow

1. Identify audience, goal, prerequisites, and expected outcome.
2. Identify whether the task is README, API docs, runbook, architecture docs, release notes, migration guide, or mixed documentation.
3. Route to a narrower skill whenever possible.
4. Inspect actual code, commands, APIs, and configuration before documenting them.
5. Keep documentation current with implementation.

## Prefer Narrow Skills

- README: `readme-writing`.
- API docs: `api-documentation`.
- Runbook: `runbook-writing`.
- Architecture docs: `architecture-docs`.
- Release notes or changelog: `release-notes`.

## Verification

- Run documented commands when practical.
- Check links, filenames, scripts, environment variables, and examples.
