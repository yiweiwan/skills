# GitHub PR Issues

## Workflow

1. Identify the PR or issue goal, audience, current status, and required action.
2. Summarize intent, user-visible behavior, implementation notes, tests, risks, and follow-ups.
3. Link related issues and decisions.
4. Respond to review comments with either a change made or a clear rationale.
5. Keep labels and issue state consistent with repository practice.

## Verification

- Check links, issue numbers, checklist items, labels, and requested reviewers.
- Do not claim CI or tests passed unless verified.
