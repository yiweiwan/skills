# Frontend Design

## Workflow

1. Inspect existing routes, components, styles, and design conventions before changing visuals.
2. Identify primary user, top task, content density, and device constraints.
3. Preserve the current design language unless the user asks for a redesign.
4. Build the real usable interface first, not a marketing explanation page.
5. Verify text, controls, and dynamic states on mobile and desktop.

## UI Rules

- Use existing component libraries, tokens, icons, spacing, and typography.
- Prefer compact, purposeful layouts for tools, SaaS, admin, CRM, and operational products.
- Use cards only for repeated items, dialogs, and genuinely framed panels.
- Avoid nested cards, decorative blobs, one-hue palettes, and oversized type in compact surfaces.

## Verification

- Run the app and inspect affected screens when possible.
- Fix clipping, overflow, overlap, layout shift, and unreadable states before finishing.
