# README Writing

## Workflow

1. Inspect the repository purpose, package scripts, dependencies, configuration, and entry points.
2. Identify the target reader: user, developer, maintainer, evaluator, or contributor.
3. Put value proposition and quick start near the top.
4. Include installation, usage, configuration, scripts, tests, deployment, troubleshooting, and contribution details when relevant.
5. Keep commands accurate and runnable.

## Structure

- Project name and one-sentence purpose.
- Features or scope.
- Quick start.
- Installation and configuration.
- Usage examples.
- Development scripts and tests.
- Contributing and license.

## Verification

- Run documented commands when practical.
- Check links, filenames, environment variables, script names, and package manager.
